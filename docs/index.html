Gradr - a webec Exercise

Project Description

This is a very simple webapplication which allows users to keep track of their grades. Users can login, see their semesters, courses, and grades. The grade average is automatically displayed foreach course and also the overall grade for the semester.

By default, I created a user in BootsTrap.groovy and a few Semesters, courses and grades belonging to that user.

Project Satisfaction

I never neither Groovy or Grails before but I thought it was quite easy to get into those. I have quite a bit of experience in web development so the overall process was nothing new.

Unfortunately I didn't have enough time to finish the project as I intended to. The login process is incomplete (no registration, profile view, etc.) and I would have loved to make a few forms a bit more user friendly. I also wasn't able to check for possible vulnerabilities.
Also I wanted to display it a bit nicer with the grades, e.g. with the finals etc. There is also no check when adding more than one final exam to the same course, which would lead to an illegal state.

Project Structure

Gradr has been implemented with the following technologies:

  • HTML
  • CSS
  • Groovy
  • Grails
  • Skeleton (Lightweight CSS Framework)

Backend

This Project is following the basic MVC pattern of the Grails framework. The following Domain classes are used for this application:

  • Course
  • Grade
  • Semester
  • User

Each of those classes have their respective controllers and services. The Logic for the calculation of the grade averages is located in
grails-app/services/GradeCalculateService.groovy.

Frontend

The Frontend is very simple. The views are located in grails-app/views. These are all .gsp files. All View files extend from the layouts/main.gsp base layout. All assets are located in grails-app/assets.

Icons

Icons are provided through icons8.com. The icons are located in src/main/resources/public/icons.

I created the Taglib IconTagLib.groovy for easy usage. E.g. for the trash icon you can just type <g:icon name="trash" /> and it will include the svg.

Development Environment

I used IntelliJ to work on this project.

start the application by running: ./grailsw run-app
test the application by running: ./grailsw test-app

The Application can be looked at by visiting localhost:8080.
